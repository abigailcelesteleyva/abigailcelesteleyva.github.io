Activity Timeline

The activity timeline lives at the bottom of the single ticket pane. It's the record of everything that happened on a ticket — who did what, when, and from where.

v2.0 · March 2026

Overview
The timeline has three actors: Source (where the ticket came from), Agent (our agent), and JK Human (any person). Plus outcomes — results with no actor.

The spine icon always shows who. The text shows what and where. Newest events first. Collapsed by default with a chevron and event count — the enrichment and suggested resolution above are the primary content.

Agent events
All use the greyed STLabs logo. "Agent assessed ticket" always shows the agent's key decisions (suggestion + assignment) as a subtle line below. "Show more" reveals the full research trail — only on this event type.
Processing
Everything the agent did on arrival collapses into one event. The agent's key decisions (suggestion + assignment) are always visible as a subtle line below. "Show more" reveals the full research trail — every bold fact that appears after "we found" in the enrichment should trace back to a specific lookup here. Format: "Queried [integration] [specific source] — [finding]". Each lookup is its own line. The enrichment narrative synthesizes what the user reported and what the agent found; the show more cites the sources behind the "we found" portion. These items should also match the integrations shown in the "Sources" bar below the enrichment.

Rules
Timestamps always include timezone (e.g., 10:41 AM PST), inline with the actor label. Field changes always show before → after — the timeline is the permanent record; the sidebar only shows the latest. Comments use a bubble background; external comments get a darker left border with the source logo inside. Status changes only appear when a human made the change or when the ticket is closed — auto-status is implied by the events that caused it.

Connecting lines run between events in the spine. The oldest event (bottom) has no line below it. "Agent assessed ticket" always shows the key decisions (suggestion + assignment) as a visible summary line. Show more only exists on this event type — it reveals the research trail. Each lookup references a specific integration + source (e.g., "Queried Okta auth logs", "Queried Workday user profile"). Every bold fact in the enrichment should trace back to a line here.
